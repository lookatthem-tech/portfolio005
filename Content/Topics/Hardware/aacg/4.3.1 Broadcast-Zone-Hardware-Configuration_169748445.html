Configuring Broadcast Zone Hardware

This Section describes how to configure video and audio for the Broadcast Zone software feature for VC (not Skype for Business) to ensure:

  • Video codec connections are properly connected and configured.
  • Prysm Application Appliance audio comes out through a speaker connected to the appliance audio port or through the display, and doesn’t get diverted to the broadcast zone device.
Note: Broadcast Zone currently does not support audio output to shared workplace users.
Note: The physical connection diagrams in this bulletin DO NOT APPLY to Prysm Solutions with Skype For Business. For information on S4B refer to About Skype for Business.
Warning: Skype for Business and an external VC codec connected to the Broadcast Zone port cannot be deployed at the same time. If a VC codec is connected, S4B must be disabled in the Web Admin.

Broadcast Zone Video Connections and Configuration For VC

Warning: When changing ANY physical connections to the Prysm Application Appliance ALWAYS do the following:
  1. Power-down/turn-off/unplug the Application Appliance and display.
  2. Modify/add physical connections.
  3. Power up the Prysm Application Appliance.

NEVER change physical connections or add hardware to the Prysm Application Appliance while the appliance is ON. Changing physical connections without shutting down the appliance may result in new hardware/connections being improperly identified by the system.

Broadcast Zone Codec Port Physical Connection Rules for All Prysm Application Appliance Models

Review these rules to understand the Broadcast Zone VC codec port connection options for your Application Appliance GPU configuration.

  1. A Broadcast Zone port is not available for a system using multiple GPUs to create the final image or mosaic. (Example: Gen1 Application Appliances in the field, with two K5000 GPUs cannot designate a Broadcast port if the IP2s/Displays are connected to both cards to create the mosaic/final image.) Note: This limitation to be addressed in the near future.
  2. One K5000 GPU (Gen 1 X40 Application Appliance, 2x DVI-DL and 2x DP ports)
    1. If DVI ports are used for IP2s/display, use the top-most DP port for Broadcast
    2. If DP ports are used for IP2s/display, use the right DVI port for Broadcast
  3. One M4000 GPU (Gen 2 X40 Application Appliance, 4x DP ports)
    • Top-most port is used for Broadcast
  4. One K1200 (X30 Application Appliance, 4 mini DP ports)
    • Top-most port is used for Broadcast. (Canvas Limiter 1.2.5.B is not required for the LCD Solutions.)
  5. One P600 GPU
    • Gen 1 X15 Application Appliance - The first Primary Display (DP) from the right side
  6. Two M4000 GPUs
    1. Gen 2 X40 Application Appliance – The left GPU must be used for both IP2/display connections only; NO BROADCAST ON THIS CARD
      • Right-most GPU's top-most port is for Broadcast Zone
    2. Gen 2 X82 Application Appliance – The right GPU must be used for both IP2/display connections only; NO BROADCAST ON THIS CARD
      • Left-most GPU's top-most port is for Broadcast Zone
  7. Two K5000 GPUs
    1. Gen 1 X40/80 Application Appliance – The left GPU must be used for both IP2/display connections only; NO BROADCAST ON THIS CARD
      • Right-most GPU's top-most port is for Broadcast Zone
    2. Gen 1 X82/86 Application Appliance – The right GPU must be used for both IP2/display connections only; NO BROADCAST ON THIS CARD
      • Left-most GPU's top-most port is for Broadcast Zone
  8. Three M4000 or three K5000 GPUs – This configuration does not support software Broadcast capability available in Synthesis 2.3+ versions since it fails rule number 1 above where the final image is most likely created with a multiple GPU mosaic.
