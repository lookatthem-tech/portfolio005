Modifying Live Source Settings in the Touch Menu

The standard Touch Menu (Top Level Menu) must be modified according to the Application Appliance Configuration or the Installation Record. This modification relates to the number, type, and naming of Live Sources, as well as the Touch Menu structure. The following steps presume that all live source capture cards were pre-provisioned by manufacturing prior to shipment.

The Touch (Top Level) Menu is the primary user interface (UI) in a Prysm Application Suite Digital Workspace. Start it by pressing and holding a single finger within an open area of a Prysm Application Suite Digital Workspace. The Touch Menu brings up from one to six menu items, with an item position defined as a numeric value (0 to 5). The 0 position is at the top of the Touch Menu; the value increments clockwise.

In the Touch (Top Level) Menu, Project Files is at position 0, Exit is at position 4, and Live Sources is at default position 3.

Live Source Mapping in the Salesforce Install and Asset Record determines where in the Touch (Top Level) Menu the Live Sources appear.

The Top Level and Friendly Name parameters are defined by Projects/AV Design; the IDs are defined by Mfg Ops; cables are defined by the Application Appliance.

Note: In the Prysm Application Suite, the Video Device Name need not correspond to the Device Name Field in Prysm Web Admin. The Device Name Field should only be populated to support legacy capture devices such as the BMD Intensity Pro. The value in Prysm Web Admin should remain Null for all Magewell devices.
  • Top Level Menu Position is the default setting (Position 3) for Live Sources in Prysm Web Admin. No more than two sublevels are allowed.
  • FriendlyName / Name can be up to 20 alpha-numeric characters in mixed case, but no special characters. It can contain main and sub-menu designators, separated by hyphens.
  • Menu Item Position is the First sub-level position set inPrysm Web Admin. No more than two sub-levels are allowed.

  • DeviceIndex is determined by matching the logical DEVICENAME of the physical card-port used for driving video input to the Index that appears when running LocalCaptureDevices. DeviceIndex increments from zero (0). Reference Information explains how to determine the DeviceIndex.
  • iPeps IP applies only to interactive sources. The device address increments from 101. When implemented in the Settings file, it contains the full form, including the local address and subnet (for example: 192.168.1.101). The tray containing the iPeps (the Interactive Input Controller) has a Prysm-factory-defined label in the form Name/iPeps IP.
  • DEVICENAME is shown when running LocalCaptureDevices.exe and is primarily for legacy system reference. Names are assigned by the Windows OS. These names cannot be changed and are used only when viewing sources in VLC (for example: determining the DeviceIndex).
  • Card-Port is the physical capture card number (prefixed by I/ to designate an input card type, as well as to differentiate it from an O/, or output, card type) and port name on that card, separated by a hyphen. Port names contain the connector type (for example: HDMI) and a number that increments from the top-down on the cards. The card number increments from 1, left to right within a chassis; it is designated by a Prysm-factory-defined label in the form I/card. Cards in an expansion chassis have card numbers prefixed with an E (for example: I/E1-HDMI).
  • Cable is defined by the label assigned to the Live Source input as connected through the System Integrator Panel (SIP). Field Staff must connect the userâ€™s Live Source inputs to the SIP in a manner consistent with the table above for Live Source Mapping.

To reset or modify the Top Level Menu and Sub-level Menu names, follow the steps below:

  1. Navigate to Web Admin > Deployments.
  2. You can either enter the MachineKey into the Search field to narrow the list until the correct Deployment appears in the list, or just select the deployment to be edited.
      1. Press Edit to go to Edit Deployment.
      2. The Edit Deployment screen displays.
  3. Press the Customizations down arrow to show options, and do the following:
    1. Select Top Level Menu.
    2. The Edit Top Level Menu Settings view displays. Scroll down to Live Sources.
  4. Open a second web browser and navigate to the Web Admin > Deployments.
    1. Find the MachineKey for the deployment you are trying to edit.
    2. Press Edit to make changes to the Deployment.
    3. Select Customizations > Live Sources to display the Edit Live Source Settings menu.
    4. Highlight and select Edit for the first Live Source to be verified.
  5. Live Source settings box displays
    1. *If the customer will be utilizing the Socket API, the IP address of the Interactive Input Controller must be entered into the Tag field.
    2. Enter the IP Address (only) from the Menu Item Param field shown in Step 6a. below into the Tag field under the Live Source settings.
  6. Go back to the Top Level Menu view and select the first Live Source to review settings.
    1. Press the Edit button to display the Live Source information.
    2. Compare the Menu Item information above to the Live Source information shown in Steps 4d. and 5 above.
    3. Make sure that the parameters and indices match.
    4. Note that in this example for Interactive Live Source, the Video Index is shown as 4, and in the End-User Top Level Menu Param tag is listed as 192.168.1.42|04.
    5. If they match, Save the information.
    6. If they did not match it would be necessary to change the Top Level Menu Param to |04 and then Save the information.
  7. Repeat this action for all Live Sources and Top Level Menu settings to confirm correct matching.
