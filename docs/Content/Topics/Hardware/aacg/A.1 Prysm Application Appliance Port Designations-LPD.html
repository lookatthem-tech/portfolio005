Prysm Application Appliance Port Designations (Custom LPD)

GPU type and quantity may vary depending on the customer requirements for the Prysm solution being installed. This Section provides examples of recommended GPU port to IP2 port connections. For more detailed information and diagrams, refer to the latest version of the Canvas Limiter Installation Guide, 520-00055-00.

Note: The X82 Gen2 Application Appliance GPU card enumerations are different than the standard left to right enumerations on the other Application Appliance models. Review the information in this section carefully to identify the correct Application Appliance configuration.

GPU Card, Port and Slot Definitions

The connection diagrams and recommendations made in Section 2 of the Canvas Limiter Installation Guide, 520-00055-00 and Configuring Canvas Limiter Scheduled Tasks in this Guide are based on the following GPU card and GPU port definitions.

FOR ALL CARDS:

  • SLOT ID – The physical slot PCI (Peripheral Component Interconnect) opening and PCI connector at the rear of the Application Appliance chassis are numbered from left to right beginning with 1, 2, 3, 4, 5, ... Slot IDs are used to identify the location of various Application Appliance cards such as a Capture Card, GPU, or NIC card. For the X40, the range is 1 to 6 and for the X82, it is 1 to 7.

FOR GPUs:

  • GPU ID – How the NVIDIA control panel identifies the card that a display is connected to (GPU 0, 1, 2). The GPU ID is dynamically assigned to a GPU when the card is connected to a display. In most cases the ID numbering starts with 0 and is sequenced from left to right. For the X40 and the X82, the range is 0 to 2.
  • GPU Logical Port Display ID – Is the logical connected display on any single GPU as viewed through the NVIDIA Control Panel. How the NVIDIA control panel identifies the port that a display is connected to (Logical Port ID 0, 1, 2, 3, 4...). The Logical Port ID is dynamically assigned to a port when it is connected to a display. The Logical Port ID numbering starts from bottom up from the first GPU to the next GPU card sequencing the "actual" display connections and begins with 0. It increments by 1 for each display connected to that single GPU (to a maximum of connected displays -1.) In the case of multiple GPUs, a connected display is completely defined by specifying both the GPU and connected display separated by a comma. For example, the third connected display to the second GPU is completely specified in the NVIDIA Control Panel as 1,2. Unused GPU ports are not included in the sequence.
  • GPU Physical Port ID – The Prysm defined GPU port identifier (Physical Port ID A, B, C, D). The Physical Port ID lettering starts with A and increments alphabetically from the bottom of the card to the top. For the M4000s it is A to D. (NVIDIA documentation refers to the physical ports numbered from 1 to 4, while the NVIDIA control panel refers to ports as 0 to 3.) For a K5000, the port IDs are also A to D but A and B are the DVI DL ports left to right respectively on the card before moving up the card to port C and D. Ports on subsequent GPUs are also identified as A, B, C, D irrespective of actual port usage.
  • Broadcast Zone Port ID: The physical connector to which a broadcast zone output (e.g., to a codec) is made. By convention, this is the last available connector on the last available GPU for a M4000. On a K5000 the recommended broadcast zone port is the DVI port on the right, GPU port ID 1 or physical port ID B. The designator is in the form GPU.PORT. For example, if two GPUs are in a system, the codec would be connected to GPU Port ID "D" on GPU ID "1" and would be denoted "1.D" . The Broadcast Zone Port ID is passed to Canvas Limiter to keep that physical port from being included in a Mosaic.
    • To run Broadcast you must be running Prysm Application Suite 2.3.1 or later and be running Canvas Limiter 1.2.5.B.
    • For an Application Appliance with 1 GPU the broadcast port is the last port.
      • X40/82 Gen 2, M4000 – GPU Card 0, GPU PORT ID D (0.D)
    • For an Application Appliance with more than 1 GPU an additional M4000 card is required for a dedicated Broadcast port.
      • X40 Gen 2, 1x M4000 and 1x M4000 – GPU Card 1, GPU PORT ID D (1.D)
      • X82 Gen 2, 2x M4000 and 1x M4000 – GPU Card 2, GPU PORT ID D (2.D)
    • Refer to the Canvas Limiter Installation Guide, 520-00055-00 for additional details on applying Broadcast correctly with Canvas Limiter.

FOR CAPTURE CARDS (CC):

  • CC Port ID: The physical connector across all capture cards (e.g. Magewell capture cards). Numbering starts with 1, on the left-most card in the system and increases from the top of the card to the bottom; where multiple cards are in a system, the CC Port ID is continuous across cards and continues to increment at the top of the next card to the immediate right.

  • Device ID: The logical designator of the capture source connected to a CC Port, and/or to a USB connector on the system. Numbering starts with 0 and increases to the maximum number (minus 1, because it is zero-ordered) of capture sources connected to the Application Appliance. There is an Audio and Video Device ID associated with every capture source. Device ID is used within the Touch Menu to bring up a live source. Device IDs are also associated with a USB-connected video camera and microphone input for use with Skype for Business.
